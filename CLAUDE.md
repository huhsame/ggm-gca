# 이 폴더는 개발 연습용 작업 공간입니다

세임이가 개발을 연습하는 공간입니다. 기술 스택과 배포 방식은 아래로 고정되어 있어요.

- **프레임워크**: Next.js (웹사이트/웹앱을 만드는 도구)
- **데이터베이스**: Supabase (회원·게시글 등 데이터를 저장하는 곳)
- **배포**: GitHub에 코드를 올리면(push) 연결된 **Vercel**이 자동으로 인터넷에 올려줍니다. 따로 배포 명령을 칠 필요 없음 — push가 곧 배포.

---

# 일하는 방식 (반드시 지킬 것)

## 1. 비개발자 눈높이로 설명
- 세임이는 컴공 전공이지만 **이 작업에서는 비개발자 기준**으로 쉽게 설명해 주세요.
- 개발 용어가 나오면 한 줄로 풀어서 같이 적기.
  - 예: "마이그레이션(= 데이터베이스 구조를 바꾸는 작업)", "환경변수(= 비밀번호 같은 설정값)", "빌드(= 코드를 실제 작동하는 형태로 조립하는 과정)"
- 영어 에러 메시지가 뜨면 **무슨 뜻인지·왜 났는지·어떻게 고치는지**를 한국어로 풀어주기.

## 2. 지금 뭘 하는지 자주 말하기
- 작업을 시작할 때, 단계가 바뀔 때마다 **"지금 ○○ 하는 중"**이라고 짧게 알려주기.
- 침묵하고 한참 작업하지 말 것. 진행 상황을 자주 중계.
- 예: "지금 로그인 화면 코드를 만들고 있어요" → "이제 화면이 잘 뜨는지 확인 중이에요" → "다 됐어요, 열어볼게요".

## 3. 시킨 것만, 단계적으로
- **사용자가 지시한 것만** 합니다. 시키지 않은 기능 추가·디자인 변경·리팩터링 금지.
- 한 번에 다 하려 하지 말고 **한 단계씩** 진행하고, 다음으로 넘어가도 되는지 흐름을 끊지 않게 진행 상황을 보여주기.
- 범위가 애매하면 멋대로 넓히지 말고 짧게 물어보기.

## 4. 할 수 있는 건 끝까지, 사람이 할 것만 골라서 안내
- 코드 작성, 파일 수정, 빌드 확인, 로컬 실행, 커밋·푸시 등 **Claude가 할 수 있는 건 최대한 직접** 끝내기.
- **사람만 할 수 있는 일**(예: 가입 인증, 브라우저에서 직접 클릭해야 하는 승인, 외부 사이트 로그인, 결제, 카드 입력 등)만 따로 모아서 **번호 매겨 알기 쉽게** 안내하기.
  - 각 안내는 "① 어디로 가서 ② 무엇을 누르고 ③ 그러면 어떻게 된다" 형태로.

## 5. 개발 끝나면 바로 확인할 수 있게 열기
- 작업이 끝나면 **항상 결과를 바로 볼 수 있게 열어** 주기.
  - 로컬에서 돌아가는 화면이면 개발 서버를 켜고 브라우저로 `http://localhost:3000` 등을 열기.
  - 배포된 결과면 Vercel URL을 열기.
  - 만든 파일/문서면 해당 파일을 열기.
- "다 됐어요"로 끝내지 말고 **눈으로 확인되는 상태까지** 만들어 주기.

# 배포 흐름 (기억해 둘 것)

1. 코드 수정 → 2. 잘 돌아가는지 확인(빌드/로컬 실행) → 3. GitHub에 push → 4. Vercel이 자동 배포 → 5. 배포된 주소 열어서 확인.

- push 전에는 가능하면 `npm run build`로 오류 없는지 먼저 확인.
- 비밀 설정값(환경변수)은 코드에 직접 쓰지 말고 `.env.local`이나 Vercel 설정에 둡니다. 키를 다시 물어보지 말고 기존 설정을 먼저 확인.
